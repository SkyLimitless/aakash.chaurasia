My tasks is to migrate our code from MVC to SOA, I started developing services for each and everything. So my primary work is focused around creating a RESTFul service’s for controller and Model. Following is the approach which I used for tasks. First, I create a Controller similar to old system but with different calls. Second, I develop a Service Implementation for Controller, which in turns call to respective Model service which it requires to access Model data in order to perform the operation. Third, I developed the Model Service which calls the Models Data Access Objects, which in turn provides the abstraction layer from the model to the outer controller. So if there is any change in Controller, one has to do no changes to the Model, it remains intact and it can also serve as Service API, which can be used for generating addition revenues. In order to test the functionality I created a Http Client which makes a REST calls and hits the Controller.
